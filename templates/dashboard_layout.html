{# dashboard_layout.html ----------------------- This is the SECOND layer of template inheritance in this project: base.html <- absolute basics: , Bootstrap CDN, our style.css/main.js (Milestone 1) -> dashboard_layout.html <- adds sidebar + navbar + footer wrapper (Milestone 4, THIS file) -> dashboard.html <- fills in just the actual page content (stat cards, charts, etc.) WHY TWO LAYERS INSTEAD OF ONE? ---------------------------------- login.html does NOT want a sidebar/navbar (it's a full-screen centered card - see Milestone 3). But every future authenticated page (Add Trainee, View Trainees, Reports...) WILL want the exact same sidebar+navbar+footer wrapper this file provides. By putting that wrapper in its own middle layer, login.html can keep extending base.html directly (skipping this file entirely), while every dashboard page extends THIS file and automatically gets the full admin-panel shell for free - we only write the sidebar/navbar/footer markup once, ever. Any page that extends this file only needs to fill in `{% block page_content %}`. #} {% extends "base.html" %} {% block extra_head %} {# Applying the saved dark-mode preference INLINE (not from dashboard.js) is deliberate: dashboard.js is a separate file the browser fetches over the network, which takes a few extra milliseconds. This tiny inline script runs synchronously while the is still being parsed, so the correct theme is set before the page paints anything at all. #} {% block dashboard_extra_head %}{% endblock %} {% endblock %} {% block content %} {# data-theme is set here (not just in CSS) so the very first paint already reflects the saved preference - dashboard.js sets this attribute from localStorage before anything below becomes visible, preventing a "flash of the wrong theme" on page load. #}
